When leader election is enabled (--ha-enable-leader-election), the InferencePool, Pod, and CRD reconcilers now also run on non-leader EPP replicas so their datastore stays populated. Non-leaders remain NotServing but this lets a request that reaches a standby EPP be routed instead of returning a 503. This is controlled by the haPopulateNonLeaderDatastore feature gate, which is enabled by default. To restore the previous leader-only behavior, disable it in the featureGates section of the EPP configuration file: `haPopulateNonLeaderDatastore=false`.
